评分模型

评分不再因为“没有搜到问题”直接给满分。系统从 72 分中位线起算，按公开证据强度加减分；具体负面样本包括网关启动失败、插件加载错误、崩溃、500、无响应、升级回归、安装修复失败、Provider 沉默和工具结果丢失。若样本不足，高分会被扣分或封顶。

默认数据源

默认使用 GitHub Releases、GitHub Issue 搜索、Release 反应数和 2026-05-05 整理的公开证据种子。若配置 window.OPENCLAW_SEARCH_API，还可接入论坛、帖子、社交媒体、新闻和网页搜索服务；点击推荐票只统计人气，不影响排名。
